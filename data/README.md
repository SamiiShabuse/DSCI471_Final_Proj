# Data

Raw Kaggle files and generated splits. **Not stored in git** (except this README).

## Layout

```
data/
├── raw/fashion-dataset/     # Kaggle download (images/, styles.csv, styles/)
└── processed/               # Generated by src/prepare_data.py
    ├── pairs.csv            # All products
    ├── train.csv / val.csv / test.csv
    ├── products.csv         # TF-IDF index
    └── *_aug.csv            # Optional - from Richardson notebooks (see below)
```

## Required setup (every clone / fresh environment)

```powershell
python src/download_kaggle_data.py   # or place Kaggle files manually
python src/prepare_data.py           # builds data/processed/
python src/prepare_data.py --check   # verify splits without rebuilding
```

`prepare_data.py` must run **before** `train.py` or `evaluate.py`. It creates splits with `product_text` (caption + JSON description) used by the TF-IDF baseline and evaluation pipeline.

## Validate existing splits

If you see errors like `missing columns ['product_text']`, your `data/processed/` files are stale (often from an old experiment copy):

```powershell
python src/prepare_data.py --check   # diagnose
python src/prepare_data.py           # fix
```

## Augmented splits (`*_aug.csv`)

Files such as `train_aug.csv` come from `notebooks/richardson_experiment/05_caption_augmentation.ipynb`. They use a **different schema** (`style` column for query-type rotation) and are only needed for Richardson's v2-v5 ablation notebooks - not for the main `src/train.py` / `src/evaluate.py` pipeline.

Do not replace `train.csv` / `val.csv` / `test.csv` with the augmented versions.
